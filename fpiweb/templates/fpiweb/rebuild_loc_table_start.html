Rebuild Location Table Start

Purpose:

This screen will add any missing records to the Location table. If there has been an increase in the number of rows, bins, and/or tiers in the warehouse area, this screen is a shortcut to adding the additional values in the appropriate tables. Change the values in the Constraints table to set the appropriate values below.

Rows:
{{ row_min }} to {{ row_max }}

Bins:
 {{ bin_min }} to {{ bin_max }}

Tiers:
 {{ tiers }}


Exclusions:
{{ exclusion_str }}


Notes:

  • This procedure will add and missing values.
  • This procedure will not delete any existing records in the Row, Bin, Tier or Location Tables.
  • This procedure will add records base on all combinations of rows, bins, and tiers given above (except for the exclusions)
  • Each record added to the Row table will have a two digit key and a description of "Row nn" where nn is the row number.
  • Each record added to the Bin table will have a two digit key and a description of "Bin nn" where nn is the bin number.
  • Each record added to the Tier table will have a two character key and a description of "Tier xx" where xx is the tier code.
  • Each record added to the Location table will have a key of "rrbbtt" and a description of "Row rr Bin bb Tier tt" (where rr = row, bb = bin and tt = tier).
